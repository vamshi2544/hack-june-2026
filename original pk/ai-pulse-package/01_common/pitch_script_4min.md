## The script

### Open · 0:00 – 0:45 · (slides 2-4)

> "Every release fails in the same window — the first hour after deploy. Not because engineers are careless, but because the failure can hide in plain sight. A deploy succeeds, health checks go green, the app reports healthy — and a downstream link silently dies. The app says it's fine. The downstream consumers just stop receiving. Nothing owns the relationship between them, so no dashboard lights up. At 2 AM on a Sunday, that gap is where response time dies.
>
> We built two agents that close it. **ReleasePilot** does the pre-deploy homework and writes a **risk packet** — what's risky about this release and what to watch for. It hands that packet to **ReleaseWatch**, which watches the deploy and catches the relationship failure the dashboards miss. The handoff is the whole idea."

*(advance to platform slide)*

### Platform + architecture · 0:45 – 1:15 · (slides 5-7) — MOVE FAST

> "And we built it for the platform Synchrony is standing up. Two agents on AgentCore Runtime, handing off via A2A — the risk packet is the payload, persisted in AgentCore Memory. Reasoning runs on **Claude through the LLM Gateway**. Data — Jira, ServiceNow, Confluence, Splunk, New Relic — comes through the MCP Gateway, all connectors already in your catalog. Humans approve through the platform's HITL primitive. Every box in our architecture is a box you already drew. Now let me show you it working."

*(switch to the demo — the live mock or deck slides 8-11)*

### Demo · 1:15 – 3:15 · the two minutes that matter

**Dashboard / L1 (1:15 – 1:35)**
> "This is ReleaseWatch's dashboard — every release it's currently watching. Nine right now, each with a risk score. This green one is credit-apply — risk 28, deployed earlier, all clear. Watch this one: application-consumer, just deployed, risk score 72 — elevated, because it touches messaging and needed a DB migration. First hour, clear. Publish rate and all three downstream consumers at baseline. The contract's holding."

**L2 drift (1:35 – 2:00)**
> "A few minutes in. application-consumer still reports healthy — green HTTP, low CPU, zero errors. But sms-consumer's inbound message rate is collapsing — eighty-five a minute down toward zero. The release notes said downstream rates wouldn't change. They're cratering. That's not a threshold breach, it's a *violated expectation.* ReleaseWatch escalates to L2 amber. Email to on-call, fifteen-minute cadence. The tile turns amber."

**L3 RCA (2:00 – 2:50) — the headline, let it breathe**
> "Minutes later, all three downstream consumers have flatlined to zero — and they did it at the exact second the deploy completed. Escalation to L3. Watch the reasoning come together. The app is healthy in isolation; three consumers are dead in isolation — together that's one relationship failure, not five symptoms. The deploy included a migration subtask — create the messaging_config table — and that migration never ran on this environment. It ran in QA, which is why QA passed. The publisher couldn't initialize against the missing schema, so it silently stopped publishing — and it only logs that at DEBUG, so nothing surfaced as an error. A past incident from March matches exactly. The agent composes all of it into one root cause — generated by Claude through the LLM Gateway — at 84 percent confidence. INC created automatically."

*(pause half a beat — this is the moment)*

**Handoff + recovery (2:50 – 3:15)**
> "Then it hands off to a human. *(Anisha clicks Acknowledge)* On-call has thirty seconds to read a structured RCA instead of ninety minutes chasing a healthy-looking app. The fix is in the RCA: run the missing migration and restart the publisher — about ten minutes, no rollback needed. *(Anisha confirms; tile turns green)* The downstream consumers come back to baseline. The watch closes. Fifty-seven minutes, detection to recovery."

### Demo vs production · 3:15 – 3:45 · (slide 12) — the honesty slide

> "One thing to be clear about. What you just saw is a TypeScript prototype of the agent experience — we vibe-coded it with GitHub Copilot so we could show you the full flow in four minutes. Production runs in Python on AgentCore, with Claude through the LLM Gateway, pulling live data through the MCP Gateway. The architecture maps one-to-one. The demo shows the behavior. The L3 prompt you saw is real — we tested it against Claude this weekend."

### Close · 3:45 – 4:00 · (slide 14)

> "Fifty-seven minutes with the agent. Three-plus hours without it — every one of us has lived that on-call. Multiply by every release and every after-hours deploy. The agent gives on-call a sixty-minute head start on the releases that go wrong. AI Pulse — ReleasePilot and ReleaseWatch. From comment to confidence."

*(stop. don't keep talking. let it land.)*

## If you're running long (cut these in order)

1. **Cut the architecture detail (slide 6)** — collapse slides 5-7 into one sentence: "Two agents on AgentCore, A2A handoff, Claude via the LLM Gateway, data via the MCP Gateway — all native to the platform."
2. **Cut the L1 beat (1:15-1:35)** — jump straight to "watch this one drift."
3. **Tighten the L3 evidence list** — say "three consumers dead at deploy time, a migration that never ran, and a matching past incident — at 84 percent confidence" instead of walking each one.

Never cut the L3 RCA moment or the demo-vs-production honesty slide. Those are the headline and the credibility.

---

## If you're running short (add these)

1. **Add the pre-deploy half** — before the dashboard, show ReleasePilot building the risk packet and the two approval gates (deck slides via the 6-8 min script). Adds ~90 seconds.
2. **Add a second past-incident detail** — "the April incident took the team two hours to diagnose manually; the agent matched it in seconds."

## Delivery notes

- **Lead with the answer, not the setup.** The first sentence states the problem, not "Hi, we're AI Pulse and we're excited to..."
- **The L3 moment is the headline.** Slow down. Pause after the confidence score. Let the room absorb it.
- **The honesty slide builds trust, not doubt.** Deliver it as a confident choice ("we vibe-coded it so we could show the full flow"), not an apology ("we didn't have time to...").
- **Stop at "from comment to confidence."** Don't trail into "so, yeah, that's our project." End on the tagline.
- **Hands off the keyboard during the handoff scene** — let Anisha physically click. It reinforces "humans stay in control."

## What NOT to say

- "We didn't have time to..." — you made deliberate scope choices; frame them that way
- "This is just a mockup" — it's a prototype of the experience; the architecture is real
- "Copilot Studio" / "Power Automate" — that's the obsolete version; we're on the Agentic AI Platform now
- "RabbitMQ ran out of memory" — that's the OLD scenario; this is a forgotten migration causing a silent publish failure
- Anything Synchrony-confidential
- Anything about a personal API key — reasoning runs on Claude via the sanctioned LLM Gateway
