Release 2026.06.08 · application-consumer · Standard Release Page v3

RP Filled from Synchrony Standard Release Page v3 by ReleasePilot · 23:14 UTC, 7 June 2026

Scope & Intent

This release adds per-consumer routing config to application-consumer's RabbitMQ publisher, so routing keys can be managed without redeploy. It introduces a new messaging_config table and requires DB migration ACQ-3852 to run before the publisher initializes. application-consumer fans out to sms-consumer, email-consumer, and audit-consumer.

▸ Risk Score
72 / 100 · Elevated — touches the messaging publish path AND requires a DB migration. Past incidents on this app cluster around messaging-init and migration-ordering failures. Migration-dependent releases are the highest-risk class for this service.

▸ Attached: INC0228994 confidence 84%
▸ Root cause
Migration subtask ACQ-3852 (create messaging_config) never ran on CDE2. The publisher (commit a7f3c19) couldn't initialize against the missing schema and silently stopped publishing — starving sms-, email-, and audit-consumer.
▸ Evidence
All 3 downstream consumers → 0/min at exact deploy time (01:32) · app HTTP healthy, 0 errors (init logs DEBUG only) · migration ACQ-3852 = NOT RUN on CDE2 (ran in QA) · matches past INC0221904
▸ Immediate fix options
(a) Run the pending messaging_config migration on CDE2, then restart the publisher — ~10 min, no rollback · or (b) roll back to build 20260601.1.18 — ~5 min. Option (a) preserves the release.
▸ Long-term
Add a publisher-health probe (not just HTTP health) to release readiness, and gate migration-dependent releases on a migration-applied check before deploy completes.
