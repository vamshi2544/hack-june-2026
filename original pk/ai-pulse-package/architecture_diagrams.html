4 · Problem Framing

Core
What it shows

The cross-system blind spot: each system reports healthy or degraded in isolation; nothing reads the relationship.

When to use

Right after the title. Establishes why the problem is hard and why per-system monitoring misses it.

"I'm healthy"
HTTP 200s · CPU 19% · 0 errors
→
publish path
silently dead
0 msg/min · init logged DEBUG only
→
sms-consumer
flatlined
inbound 0/min · baseline ~85/min
Per-system monitoring sees
3 separate dashboards · 3 separate thresholds · no single alert that says "these are one event"
vs
ReleaseWatch sees
one relationship failure: a forgotten migration silently killed the publish path, predicted by the risk packet, matching past INC0221904
At 2 AM, an on-call engineer stares at a green app and pieces this together in 90+ minutes. The agent reads the app against its downstream consumers and the release intent, and names the relationship in seconds.

Without the handoff, ReleaseWatch is generic threshold monitoring. With it, every threshold is calibrated to what this specific release intended to change. The risk packet is the contract.
